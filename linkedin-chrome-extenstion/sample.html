    👉 Excited to showcase my latest project: Real-time Click Stream Data Analysis using AWS!

    👉 Tools Used - Python Flask & AWS services - Kinesis, Glue, Aapche flink, Zipline.

    Batch processing vs. Stream processing
    👉 Batch processing involves the accumulation of data over a period, which is then analyzed and processed for utilization. This method is similar to how we watch movies on OTT platforms like Hotstar, Netflix, and Amazon Prime.

    👉On the other hand, stream processing involves collecting, analyzing, and processing data simultaneously or in smaller, sequential pieces as it is being collected. A best example of stream processing is with the live streaming of cricket matches on platforms like Hotstar, where data is processed in real-time or in smaller increments as it arrives.

    👉Similarly, let's consider an E-commerce site aiming to identify which products are being clicked how many times during a live sale. This real-time tracking enables the platform to assess user interests and adjust their stock accordingly for products like mobiles, laptops, and televisions in this case.

    👉This stream processing of product clicks was analyzed with the assistance of AWS Kinesis and Apache Flink. The real-time output data was then visually examined using AWS Zipline.
